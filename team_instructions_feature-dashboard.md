# Instrucciones para el equipo de Dashboard

Rama: feature-dashboard
Base: Deploy
Propósito: Implementar el tablero (dashboard) de métricas y vistas principales del sistema.

Flujo de trabajo recomendado:
- Trabajen directamente en esta rama `feature-dashboard`.
- Hagan commits pequeños y descriptivos: `feat(dashboard): agregar componente de métricas`.
- Mantengan el código compilable y con tests pasando antes de pushear.
- Al terminar una funcionalidad o un conjunto de cambios coherente, abran un Pull Request hacia `Deploy` (base: `Deploy`, compare: `feature-dashboard`).

Checklist antes de abrir PR:
- [ ] Lint y formateo aplicados
- [ ] Tests relevantes añadidos/actualizados
- [ ] Documentación o comentarios aclaratorios donde aplique
- [ ] Descripción del PR con capturas o pasos para probar
